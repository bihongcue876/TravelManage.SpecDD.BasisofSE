# SSD 规范设计文档 (language: zh-CN)

## 1. applications.feature - 旅游申请管理

```gherkin
# language: zh-CN
功能: 旅游申请管理
  作为前台员工
  我想要为顾客办理旅游申请
  以便于确认团位并收取订金

  背景:
    假如 系统中存在旅游路线 "武汉-三亚3日游"，路线ID为 1
    而且 系统中存在旅游团，团ID为 1，团代码 "WH-SY-001"
    而且 该团属于路线 1
    而且 该团出发日期为 "2026-08-20"
    而且 该团申请截止日期为 "2026-08-10"
    而且 该团人数限额为 30 人
    而且 该团大人价格为 2000 元
    而且 该团小孩价格为 1000 元
    而且 该团已发布

  场景: 成功创建旅游申请 - 距出发超过两个月
    假如 今天是 "2026-05-13"
    当 员工为顾客创建申请:
      | 字段     | 值           |
      | 团ID     | 1            |
      | 姓名     | 张三         |
      | 电话     | 13800138000  |
      | 邮箱     | zhang@test.com |
      | 地址     | 武汉市洪山区 |
      | 邮编     | 430000       |
      | 大人数量 | 2            |
      | 小孩数量 | 1            |
    那么 申请创建成功
    而且 申请状态为 "draft"
    而且 订金为 500 元
    而且 总价为 5000 元
    而且 订金比例为 "10%"

  场景: 成功创建旅游申请 - 距出发一个月到两个月之间
    假如 今天是 "2026-07-01"
    当 员工为顾客创建申请:
      | 字段     | 值           |
      | 团ID     | 1            |
      | 姓名     | 李四         |
      | 电话     | 13900139000  |
      | 大人数量 | 1            |
      | 小孩数量 | 0            |
    那么 申请创建成功
    而且 订金为 400 元
    而且 订金比例为 "20%"

  场景: 成功创建旅游申请 - 距出发不足一个月需付全款
    假如 今天是 "2026-07-25"
    当 员工为顾客创建申请:
      | 字段     | 值           |
      | 团ID     | 1            |
      | 姓名     | 王五         |
      | 电话     | 13700137000  |
      | 大人数量 | 1            |
      | 小孩数量 | 0            |
    那么 申请创建成功
    而且 订金为 2000 元
    而且 订金比例为 "100%"

  场景: 创建申请失败 - 旅游团未发布
    假如 该团未发布
    当 员工为顾客创建申请:
      | 字段     | 值           |
      | 团ID     | 1            |
      | 姓名     | 赵六         |
      | 电话     | 13600136000  |
      | 大人数量 | 1            |
      | 小孩数量 | 0            |
    那么 申请创建失败
    而且 错误信息为 "Group is not published"

  场景: 创建申请失败 - 已过申请截止日期
    假如 今天是 "2026-08-11"
    当 员工为顾客创建申请:
      | 字段     | 值           |
      | 团ID     | 1            |
      | 姓名     | 孙七         |
      | 电话     | 13500135000  |
      | 大人数量 | 1            |
      | 小孩数量 | 0            |
    那么 申请创建失败
    而且 错误信息为 "Application deadline has passed"

  场景: 创建申请失败 - 旅游团人数已满
    假如 该团当前已报名 30 人
    当 员工为顾客创建申请:
      | 字段     | 值           |
      | 团ID     | 1            |
      | 姓名     | 周八         |
      | 电话     | 13400134000  |
      | 大人数量 | 1            |
      | 小孩数量 | 0            |
    那么 申请创建失败
    而且 错误信息为 "Not enough spots available"

  场景: 创建申请失败 - 旅游团人数不足
    假如 该团当前已报名 29 人
    当 员工为顾客创建申请:
      | 字段     | 值           |
      | 团ID     | 1            |
      | 姓名     | 吴九         |
      | 电话     | 13300133000  |
      | 大人数量 | 2            |
      | 小孩数量 | 0            |
    那么 申请创建失败
    而且 错误信息为 "Not enough spots available"

  场景: 创建申请失败 - 旅游团不存在
    当 员工为顾客创建申请:
      | 字段     | 值           |
      | 团ID     | 999          |
      | 姓名     | 郑十         |
      | 电话     | 13200132000  |
      | 大人数量 | 1            |
      | 小孩数量 | 0            |
    那么 申请创建失败
    而且 错误信息为 "Group not found"

  场景: 支付订金成功
    假如 存在申请ID为 1，状态为 "draft"，订金为 500 元
    当 员工为该申请支付订金 500 元
    那么 支付成功
    而且 申请状态变为 "deposit_paid"
    而且 已支付订金为 500 元
    而且 生成支付记录，类型为 "deposit"，金额为 500 元

  场景: 支付订金失败 - 申请状态不正确
    假如 存在申请ID为 1，状态为 "deposit_paid"
    当 员工为该申请支付订金 500 元
    那么 支付失败
    而且 错误信息为 "Application is not in draft state"

  场景: 支付订金失败 - 订金已支付
    假如 存在申请ID为 1，状态为 "draft"，已支付订金 500 元
    当 员工为该申请支付订金 500 元
    那么 支付失败
    而且 错误信息为 "Deposit already paid"

  场景: 支付订金失败 - 支付金额不足
    假如 存在申请ID为 1，状态为 "draft"，订金为 500 元
    当 员工为该申请支付订金 400 元
    那么 支付失败
    而且 错误信息为 "Deposit amount is insufficient"

  场景: 录入参加者信息成功
    假如 存在申请ID为 1，状态为 "deposit_paid"，大人2名，小孩1名
    当 员工录入参加者信息:
      | 姓名   | 性别 | 出生日期    | 电话       | 是否负责人 |
      | 张三   | M    | 1980-01-01 | 13800138000| 是         |
      | 李四   | F    | 1985-05-15 | 13900139000| 否         |
      | 张小明 | M    | 2010-03-20 | 13700137000| 否         |
    那么 录入成功
    而且 申请信息已完成
    而且 参加者数量为 3 人

  场景: 录入参加者失败 - 人数不匹配
    假如 存在申请ID为 1，状态为 "deposit_paid"，大人2名，小孩1名
    当 员工录入参加者信息:
      | 姓名   | 性别 | 出生日期    | 是否负责人 |
      | 张三   | M    | 1980-01-01 | 是         |
      | 李四   | F    | 1985-05-15 | 否         |
    那么 录入失败
    而且 错误信息为 "Expected 3 participants, got 2"

  场景: 录入参加者失败 - 缺少负责人
    假如 存在申请ID为 1，状态为 "deposit_paid"，大人2名，小孩0名
    当 员工录入参加者信息:
      | 姓名   | 性别 | 出生日期    | 是否负责人 |
      | 张三   | M    | 1980-01-01 | 否         |
      | 李四   | F    | 1985-05-15 | 否         |
    那么 录入失败
    而且 错误信息为 "At least one participant must be the leader"

  场景: 录入参加者失败 - 申请已取消
    假如 存在申请ID为 1，状态为 "cancelled"
    当 员工录入参加者信息:
      | 姓名   | 性别 | 出生日期    | 是否负责人 |
      | 张三   | M    | 1980-01-01 | 是         |
    那么 录入失败
    而且 错误信息为 "Application is cancelled"

  场景: 支付尾款成功 - 部分支付
    假如 存在申请ID为 1，状态为 "deposit_paid"，信息已完成
    而且 总价为 5000 元，已付订金 500 元，已付尾款 0 元
    当 员工支付尾款 2000 元
    那么 支付成功
    而且 已付尾款为 2000 元
    而且 申请状态仍为 "deposit_paid"

  场景: 支付尾款成功 - 付清全款
    假如 存在申请ID为 1，状态为 "deposit_paid"，信息已完成
    而且 总价为 5000 元，已付订金 500 元，已付尾款 0 元
    当 员工支付尾款 4500 元
    那么 支付成功
    而且 已付尾款为 4500 元
    而且 申请状态变为 "confirmed"

  场景: 支付尾款失败 - 信息未完成
    假如 存在申请ID为 1，状态为 "deposit_paid"，信息未完成
    当 员工支付尾款 1000 元
    那么 支付失败
    而且 错误信息为 "Participant information is not completed"

  场景: 支付尾款失败 - 申请已取消
    假如 存在申请ID为 1，状态为 "cancelled"
    当 员工支付尾款 1000 元
    那么 支付失败
    而且 错误信息为 "Application is cancelled"

  场景: 支付尾款失败 - 支付金额超出剩余尾款
    假如 存在申请ID为 1，状态为 "deposit_paid"，信息已完成
    而且 总价为 5000 元，已付订金 500 元，已付尾款 0 元
    当 员工支付尾款 5000 元
    那么 支付失败
    而且 错误信息为 "Payment amount exceeds remaining balance"

  场景: 查询申请成功
    假如 存在申请ID为 1
    当 员工查询申请详情，申请ID为 1
    那么 查询成功
    而且 返回申请详细信息，包括参加者列表和旅游团信息

  场景: 查询申请失败 - 申请不存在
    当 员工查询申请详情，申请ID为 999
    那么 查询失败
    而且 错误信息为 "Application not found"

  场景: 搜索申请 - 按团代码搜索
    假如 存在多个申请，分别属于不同旅游团
    当 员工按团代码 "WH-SY-001" 搜索申请
    那么 返回该团的所有申请

  场景: 搜索申请 - 按负责人姓名搜索
    假如 存在多个申请，负责人姓名不同
    当 员工按姓名 "张三" 搜索申请
    那么 返回负责人姓名包含 "张三" 的所有申请

  场景: 搜索申请 - 按出发日期范围搜索
    假如 存在多个申请，出发日期不同
    当 员工按出发日期范围 "2026-08-01" 到 "2026-08-31" 搜索申请
    那么 返回出发日期在该范围内的所有申请

  场景: 更新参加者信息成功
    假如 存在参加者ID为 1，姓名为 "张三"
    当 员工更新参加者信息:
      | 字段   | 值           |
      | 姓名   | 张三丰       |
      | 电话   | 13800138001  |
    那么 更新成功
    而且 参加者姓名为 "张三丰"
    而且 参加者电话为 "13800138001"

  场景: 更新参加者失败 - 参加者不存在
    当 员工更新参加者信息，参加者ID为 999:
      | 字段   | 值           |
      | 姓名   | 张三丰       |
    那么 更新失败
    而且 错误信息为 "Participant not found"
```

## 2. groups.feature - 旅游团管理

```gherkin
# language: zh-CN
功能: 旅游团管理
  作为路线管理员
  我想要管理旅游团信息
  以便于为顾客提供旅游产品

  背景:
    假如 系统中存在旅游路线 "武汉-三亚3日游"，路线ID为 1

  场景: 成功创建旅游团
    当 管理员创建旅游团:
      | 字段         | 值           |
      | 路线ID       | 1            |
      | 团代码       | WH-SY-001    |
      | 出发日期     | 2026-08-20   |
      | 截止日期     | 2026-08-10   |
      | 人数限额     | 30           |
      | 大人价格     | 2000         |
      | 小孩价格     | 1000         |
    那么 旅游团创建成功
    而且 旅游团状态为 "未发布"
    而且 旅游团代码为 "WH-SY-001"

  场景: 创建旅游团失败 - 路线不存在
    当 管理员创建旅游团:
      | 字段         | 值           |
      | 路线ID       | 999          |
      | 团代码       | WH-SY-002    |
      | 出发日期     | 2026-08-20   |
      | 截止日期     | 2026-08-10   |
      | 人数限额     | 30           |
    那么 创建失败
    而且 错误信息为 "Route not found"

  场景: 创建旅游团失败 - 截止日期晚于出发日期
    当 管理员创建旅游团:
      | 字段         | 值           |
      | 路线ID       | 1            |
      | 团代码       | WH-SY-003    |
      | 出发日期     | 2026-08-10   |
      | 截止日期     | 2026-08-20   |
      | 人数限额     | 30           |
    那么 创建失败
    而且 错误信息为 "Deadline must be before departure date"

  场景: 更新旅游团成功 - 未发布状态
    假如 存在旅游团ID为 1，状态为 "未发布"
    当 管理员更新旅游团信息:
      | 字段         | 值           |
      | 团代码       | WH-SY-001-UPD|
      | 人数限额     | 35           |
      | 大人价格     | 2200         |
      | 小孩价格     | 1100         |
    那么 更新成功
    而且 旅游团人数限额为 35 人
    而且 大人价格为 2200 元

  场景: 更新旅游团失败 - 已发布状态
    假如 存在旅游团ID为 1，状态为 "已发布"
    当 管理员更新旅游团信息:
      | 字段         | 值           |
      | 人数限额     | 40           |
    那么 更新失败
    而且 错误信息为 "Cannot modify published group"

  场景: 发布旅游团成功
    假如 存在旅游团ID为 1，状态为 "未发布"
    而且 大人价格为 2000 元，小孩价格为 1000 元
    当 管理员发布旅游团
    那么 发布成功
    而且 旅游团状态为 "已发布"

  场景: 发布旅游团失败 - 已发布
    假如 存在旅游团ID为 1，状态为 "已发布"
    当 管理员发布旅游团
    那么 发布失败
    而且 错误信息为 "Group is already published"

  场景: 发布旅游团失败 - 价格未设置
    假如 存在旅游团ID为 1，状态为 "未发布"
    而且 大人价格未设置
    当 管理员发布旅游团
    那么 发布失败
    而且 错误信息为 "Prices must be set before publishing"

  场景: 查询旅游团列表 - 全部
    假如 系统中存在多个旅游团
    当 管理员查询所有旅游团
    那么 返回所有旅游团列表
    而且按出发日期排序

  场景: 查询旅游团列表 - 仅已发布
    假如 系统中存在多个旅游团，部分已发布，部分未发布
    当 管理员查询已发布的旅游团
    那么 返回所有已发布的旅游团列表

  场景: 查询旅游团列表 - 按路线筛选
    假如 系统中存在多个旅游团，属于不同路线
    当 管理员按路线ID 1 查询旅游团
    那么 返回该路线的所有旅游团

  场景: 查询旅游团列表 - 按出发日期范围筛选
    假如 系统中存在多个旅游团，出发日期不同
    当 管理员按出发日期范围 "2026-08-01" 到 "2026-08-31" 查询旅游团
    那么 返回出发日期在该范围内的旅游团

  场景: 查询旅游团详情成功
    假如 存在旅游团ID为 1
    当 管理员查询旅游团详情，团ID为 1
    那么 查询成功
    而且返回旅游团详细信息，包括路线信息

  场景: 查询旅游团详情失败 - 旅游团不存在
    当 管理员查询旅游团详情，团ID为 999
    那么 查询失败
    而且 错误信息为 "Group not found"

  场景: 检查旅游团可用名额 - 有剩余名额
    假如 存在旅游团ID为 1，人数限额 30 人
    而且 当前已报名 20 人
    当 管理员检查旅游团可用名额
    那么 返回可用名额信息:
      | 字段     | 值    |
      | 人数限额 | 30    |
      | 已报名   | 20    |
      | 剩余名额 | 10    |

  场景: 检查旅游团可用名额 - 已满
    假如 存在旅游团ID为 1，人数限额 30 人
    而且 当前已报名 30 人
    当 管理员检查旅游团可用名额
    那么 返回可用名额信息:
      | 字段     | 值    |
      | 人数限额 | 30    |
      | 已报名   | 30    |
      | 剩余名额 | 0     |

  场景: 检查旅游团可用名额 - 有取消申请
    假如 存在旅游团ID为 1，人数限额 30 人
    而且 当前已报名 30 人
    而且 有 2 人取消申请
    当 管理员检查旅游团可用名额
    那么 返回可用名额信息:
      | 字段     | 值    |
      | 人数限额 | 30    |
      | 已报名   | 28    |
      | 剩余名额 | 2     |

  场景: 预览订金计算 - 距出发超过两个月
    假如 存在旅游团ID为 1，出发日期 "2026-08-20"
    而且 大人价格 2000 元，小孩价格 1000 元
    假如 今天是 "2026-05-13"
    当 管理员预览订金计算，大人 2 名，小孩 1 名
    那么 返回价格预览:
      | 字段       | 值      |
      | 订金       | 500     |
      | 总价       | 5000    |
      | 订金比例   | 10%     |

  场景: 预览订金计算 - 距出发一个月到两个月之间
    假如 存在旅游团ID为 1，出发日期 "2026-08-20"
    而且 大人价格 2000 元，小孩价格 1000 元
    假如 今天是 "2026-07-01"
    当 管理员预览订金计算，大人 2 名，小孩 1 名
    那么 返回价格预览:
      | 字段       | 值      |
      | 订金       | 1000    |
      | 总价       | 5000    |
      | 订金比例   | 20%     |

  场景: 预览订金计算 - 距出发不足一个月
    假如 存在旅游团ID为 1，出发日期 "2026-08-20"
    而且 大人价格 2000 元，小孩价格 1000 元
    假如 今天是 "2026-07-25"
    当 管理员预览订金计算，大人 2 名，小孩 1 名
    那么 返回价格预览:
      | 字段       | 值      |
      | 订金       | 5000    |
      | 总价       | 5000    |
```
